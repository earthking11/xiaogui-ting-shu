# 更新日志

## v2 - 2026-06-18

这是一次大版本更新，重点新增“整本听书 / 歌词式听书”，并修复生成、播放、滚动和存储一致性问题。

### 整本听书

- 首页新增 `整本听书` 入口，不覆盖原来的阅读和即时 TTS 听书方式。
- 支持先把整本 TXT 逐句生成 MP3，完成后像听歌看歌词一样播放。
- 播放时当前句会高亮，用户可以点击任意一句直接跳到对应音频。
- 播放会自动续播下一句，并让页面跟随当前高亮句滚动。
- 整本生成支持进度显示、暂停、继续、失败重试和本地缓存。
- TTS 生成改为 MP3，并支持 2 路并发请求，加快整本生成速度。

### 体验优化

- 整本生成卡片支持收起，收起后只保留轻量进度条和核心操作，减少阅读遮挡。
- 修复点击中间句后不会继续播放下一句的问题。
- 修复自动切到下一句时页面可能先往上再往下、甚至跳回错误段落的问题。
- 底部播放器显示当前句序号和句内播放进度。
- 首页会显示整本听书生成状态，例如 `可播放 · 101/101 句`。

### 稳定性与数据一致性

- 修复生成任务中残留 `generating` 状态导致“明明没生成完却显示完成”的问题。
- 修复删除整本音频时可能和正在生成的后台任务互相覆盖的问题。
- 修复退出整本听书页时播放器回调可能访问已释放播放器的崩溃风险。
- 修复最后一句播放结束后再次点击播放行为不确定的问题，现在会从第一句重新开始。
- 切换整本听书生成方案时会清理同一本书旧 plan，减少本地 MP3 长期堆积。
- 没有正在生成时调用暂停不再把已完成状态误改成暂停。

### 验证

- 已用 101 句测试文本验证整本生成、点击中间句跳播、自动续播和页面跟随滚动。
- `dart analyze .` 通过。
- `flutter test` 通过，22 个测试全绿。
- Android arm64 APK 构建通过。

## v1.1 - 2026-06-12

这是一次体验打磨版本，重点优化阅读页、朗读设置、错误反馈和进度恢复。

### 阅读体验

- 底部工具栏中间入口从 `Aa` 改为 `阅读及 TTS设置`，入口含义更清楚。
- 阅读页底部留白会根据工具栏和听书播放器自动调整，减少正文被遮挡。
- 右下角阅读百分比在工具栏或听书播放器出现时会自动上移。
- 强杀 App 后重新进入，会按实际可见位置重新计算阅读百分比，避免短书场景下百分比和页面位置不一致。
- 书签提示更明确，会显示当前已读百分比；重复书签提示改得更自然。
- 阅读页提示条会根据底部播放器和工具栏上移，不再压住底部操作区。

### 阅读及 TTS 设置

- 设置页标题改为 `阅读及 TTS 设置`。
- 设置页拆分为 `阅读外观`、`朗读设置`、`MiMo API Key`，结构更清楚。
- 主题、字号、行距、音色、语速现在点击后会立刻刷新选中态，不再像“点了没反应”。
- `保存 Key` 在输入为空时会立刻显示提示。
- 保存、清除和连通性测试结果改为页内反馈卡片，并用颜色区分成功和失败。
- API Key 输入框继续使用普通文本输入，避免触发部分安卓系统的安全键盘。

### 听书与错误反馈

- 首次听书弹出的 API Key 输入框也改为普通文本输入，不再触发安全键盘。
- 听书播放器按钮文案改为更短的 `暂停`、`继续`、`重试`、`填 Key`，小屏显示更稳。
- 假 Key 或认证失败时，播放器会提示 `请检查 Key 或网络后重试`，不再误显示 `正在生成快速起播段`。
- 更多面板中的朗读状态改为中文，例如 `正在朗读`、`准备下一段`、`出错`，不再显示开发者状态名。
- 更多面板中的 Key 入口改为 `朗读与 Key 设置`，并只显示掩码后的 Key。

### 验证

- 已在 Android 模拟器中验证 TXT 导入、阅读、设置即时反馈、空 Key 提示、假 Key 连通性错误、假 Key 听书错误、书签提示、强杀恢复进度。
- `flutter analyze` 通过。
- TTS 相关测试通过。
- Android arm64 debug APK 构建通过。

## v1.0 - 2026-06-12

首个可用测试版本。

- 支持导入本地 TXT 小说。
- 支持从系统文件选择器读取 TXT。
- 支持阅读进度恢复、主题、字号、行距、书签。
- 支持填写 MiMo API Key。
- 支持 MiMo TTS 分段生成与播放。
- 支持快速起播段和下一段预生成。
- 支持底部听书播放器和右下角阅读百分比。
