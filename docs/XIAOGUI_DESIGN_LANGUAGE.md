# 小龟系列设计语言

适用产品：小龟听书、小龟寻物，以及后续以“小龟”为主角的轻量工具型应用。

## 1. 品牌气质

小龟系列的核心是“温柔、有用、不过度打扰”。小龟不是卡通装饰，而是功能的引导者：读书时安静陪读，寻物时帮助发现。整体应保持亲和、可信、轻盈，避免高饱和的游戏化界面和冰冷的效率工具感。

## 2. 应用图标

| 项目 | 规范 |
| --- | --- |
| 图标结构 | 暖白圆角方形底，中央为 3D 柔光小龟与任务物件。 |
| 角色比例 | 小龟与核心物件构成视觉主体，建议占图标安全画布高度的 52%-58%，与同系列图标的角色高度保持一致。 |
| 前景视觉占比 | Android `432 x 432` 前景图中，背景卡片可延伸至画布边缘并交给系统蒙版裁切；角色本体不以卡片大小代替校准。 |
| 安全边距 | 小龟、书本、放大镜等核心内容四周至少保留约 `12%` 安全区；允许外层卡片与柔光延伸到边缘。 |
| 圆角 | 由系统自适应图标蒙版决定；插画内卡片使用大圆角，避免硬直角。 |
| 内容变化 | 保留同一只小龟、暖白底与柔光质感；只替换任务物件，例如书本、放大镜、钥匙。 |
| 密度资源 | 始终同步输出 `mdpi`、`hdpi`、`xhdpi`、`xxhdpi`、`xxxhdpi` 的普通、圆形与前景图标。 |

小龟听书与小龟寻物的图标应在同一桌面网格中呈现相同的视觉重量，而不是只让 PNG 像素尺寸相同。

## 3. 色彩

### 系列基础色

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 墨绿文字 | `#15201C` | 默认标题、正文和高对比信息。 |
| 云雾白 | `#F6FAF7` | 主页面、输入区域与轻背景。 |
| 龟壳薄荷绿 | `#2F9E7E` | 主操作、成功状态、可交互焦点。 |
| 深薄荷绿 | `#1C6F5B` | 按下态、深色文字与强调层级。 |
| 暖琥珀 | `#F2A950` | 进度、提示、任务重点和少量温度点缀。 |
| 柔和分隔线 | `#DDE8E1` | 卡片边界、输入框与低优先级分隔。 |

### 产品主题扩展

小龟寻物以薄荷绿为主色，表达发现与整理。小龟听书保留纸张、暖黄、鼠尾草绿与夜间主题，表达长时间阅读的舒适性；其默认纸白主题的重点色为 `#8D5E42`，不应与寻物的薄荷主操作产生冲突。

## 4. 排版

| 层级 | 建议 | 已有实践 |
| --- | --- | --- |
| 页面主标题 | 28-34sp，700 字重，紧凑行高 | 听书书架标题。 |
| 区块标题 | 18-22sp，600-700 字重 | 卡片与设置分组。 |
| 正文 | 15-18sp，常规字重，阅读内容可放大 | 听书阅读页。 |
| 辅助信息 | 13-14sp，中低对比 | 时间、状态、说明。 |

中文排版优先保证行距和字色对比。阅读型页面允许大字号和宽行距；工具型页面优先保持信息扫描效率。

## 5. 形状与层级

- 卡片使用 18-30dp 的大圆角，营造“柔软纸片”而非仪表盘。
- 主要按钮与状态胶囊使用全圆角或 22dp 以上圆角。
- 背景以低对比浅色、轻微渐变或极淡纹理组织层级；不使用重阴影堆叠。
- 分隔优先使用 `#DDE8E1` 一类浅线；阴影只用于悬浮控件与底部浮层。
- 底部弹层采用 30dp 顶部圆角，并保持清晰标题和足够触控面积。

## 6. 交互反馈

- 每个可点控件必须有按压反馈、禁用态或结果提示，不能只在数据层生效。
- 主操作使用明确动词，例如“导入 TXT”“开始听书”“开始识别”。
- 长任务必须展示阶段、进度、暂停/继续或可恢复状态；不能让用户面对无解释的等待。
- 朗读、识别、导入等状态优先使用安静的文字和进度，而不是频繁弹窗。

## 7. 无障碍与质量底线

- 正文与背景保持足够对比，辅助文字不可使用过淡灰色。
- 关键操作最小触控区域建议不低于 44dp。
- 颜色不作为唯一状态信号，需同时提供图标、文字或进度信息。
- 深色模式沿用同一层级关系：暖色/薄荷色降低亮度，不使用刺眼纯白。

## 8. 新产品落地清单

1. 从“同一只小龟 + 新任务物件”定义图标，而不是重画一套品牌。
2. 先选薄荷工具调或纸张阅读调，再确定单一主操作色。
3. 使用大圆角、浅背景、克制阴影和高可读中文排版。
4. 为所有等待、失败、完成与恢复流程补齐可见反馈。
5. 上线前在真实桌面网格中与已有两款图标并排检查视觉重量。
